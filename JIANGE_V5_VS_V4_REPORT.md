## 二、架构对比

### v4.0 架构

```
单进程脚本
    ↓
正则匹配关键词
    ↓
直接输出报告
```

**特点**:
- 单进程顺序处理
- 简单正则匹配
- 无持久化存储
- 无工作流管理
- 一次性脚本，不可复用

### v5.0 架构

```
Gateway (CLI/WebSocket/Feishu)
    ↓
Scheduler (优先级队列)
    ↓
Skill System (强制工作流)
    ↓
Agent Core (并行分析)
    ↓
PalaceV2 (分层存储)
```

**特点**:
- 5层架构，职责分离
- 强制工作流: read→extract→verify→store→report
- 并行分析: 4维度同时提取 (Character/Plot/Pacing/Catharsis)
- 分层存储: L0(原始)→L1(结构化)→L2(语义索引)
- 图遍历检索 (Tunnel)
- 可扩展技能系统
